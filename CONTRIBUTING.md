# Contributing to Horizon Frontend

Thank you for your interest in contributing to Horizon! We welcome contributions from the community to make this project better.

## Getting Started

1. **Fork the repository**
2. **Clone your fork locally**
   ```bash
   git clone https://github.com/awscloudclublpu/horizon_frontend.git
   cd horizon_frontend
   ```
3. **Install dependencies**
   ```bash
   npm install
   # or
   yarn install
   # or
   pnpm install
   ```
4. **Set up environment variables**
   - Copy `.env.example` to `.env` and fill in the required values.

5. **Create a local build**
   ```bash
   npm run build
   # or
   yarn build
   # or
   pnpm build
   ```
   - Ensure the build completes successfully before submitting a PR.

6. **Run the development server**
   ```bash
   npm run dev
   # or
   yarn dev
   # or
   pnpm dev
   ```

## Creating a Pull Request (PR)

- Make sure your branch is up to date with `main`.
- Ensure your code follows the project's style and passes linting.
- Test your changes locally.
- Submit your PR with a clear description of your changes.
- Reference any related issues in your PR description.

## Code Style & Guidelines

- Use TypeScript and Next.js best practices.
- Follow the existing folder and component structure.
- Use Tailwind CSS for styling.
- Keep components modular and reusable.
- Add comments where necessary for clarity.

## Reporting Issues

- Use the GitHub Issues tab to report bugs or request features.
- Provide as much detail as possible, including steps to reproduce.
